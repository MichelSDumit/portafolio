Arduino UNO Electrónica Bluetooth

- Resumen -

En esta práctica, nos enfocamos en el uso del módulo HC-05 para establecer una comunicación inalámbrica mediante Bluetooth entre un Arduino UNO y una aplicación móvil. Este tipo de conexión permite la transmisión de datos de manera sencilla y eficiente, facilitando el control remoto de dispositivos electrónicos.

Para el desarrollo de la aplicación, utilizamos la plataforma en línea MIT App Inventor, la cual nos permitió diseñar una interfaz gráfica intuitiva y funcional. A través de esta aplicación, logramos enviar comandos al Arduino, lo que nos permitió controlar el encendido y apagado de un LED en tiempo real mediante la conexión Bluetooth.

- Introducción -

El Bluetooth es una tecnología de comunicación inalámbrica de corto alcance que permite la conexión directa entre dos dispositivos sin necesidad de una infraestructura de red adicional, como un enrutador o un punto de acceso.

Mediante una aplicación móvil, es posible controlar diferentes funciones de un sistema de forma remota, aprovechando la conexión Bluetooth para enviar y recibir datos en tiempo real. Esto facilita la interacción con el sistema a través de una interfaz personalizada, optimizando su uso y mejorando la experiencia del usuario.

- Materiales -

1. Placa Arduino UNO.

2. Cable USB para la conexión con el ordenador.

3. Módulo Bluetooth HC05.

- Desarrollo -

Para desarrollar la aplicación, utilizamos la plataforma MIT App Inventor, la cual facilita la creación de aplicaciones mediante una interfaz visual basada en bloques. Esta metodología permite diseñar y programar de manera intuitiva, sin necesidad de escribir código de forma tradicional.

A través de estos bloques, es posible ejecutar diversos procesos, como la captura de datos en la aplicación y su posterior transmisión al Arduino mediante la conexión Bluetooth. Esto simplifica la comunicación entre la app y el microcontrolador, permitiendo un control eficiente del sistema.

Diseño

La aplicación cuenta con cinco botones principales: uno para conectarse a un dispositivo Bluetooth, otro para desconectarse, uno para encender el LED, otro para apagarlo y un último botón para cerrar la aplicación.

Una vez iniciada la aplicación y establecida la conexión con el módulo HC-05, los botones de encendido y apagado envían los valores '1' y '0' respectivamente. El Arduino, mediante su programación, interpreta estas señales y ajusta el estado del LED en función del último comando recibido, manteniéndolo encendido o apagado según corresponda.

Construcción

La construcción del circuito es sencilla, ya que únicamente requiere la conexión del módulo HC-05 al Arduino UNO. Para ello, se deben conectar los pines de alimentación del módulo (VCC y GND) a los pines correspondientes en el Arduino, asegurando un suministro estable de energía.

Además, los pines de comunicación del módulo HC-05, TX y RX, deben vincularse a los pines adecuados del Arduino para establecer la transmisión de datos serie. En algunos casos, es recomendable utilizar un divisor de voltaje en la línea RX del módulo para evitar posibles daños, ya que el HC-05 opera a 3.3V en su entrada de datos, mientras que el Arduino trabaja a 5V.

- Resultados -

La aplicación logró controlar el LED de manera exitosa, permitiéndonos establecer una conexión estable con el Arduino y enviar comandos a través de los botones de la interfaz. Gracias a esto, pudimos encender y apagar el LED de forma remota mediante la comunicación Bluetooth, demostrando el correcto funcionamiento del sistema.

- Conclusiones -

Se desarrolló exitosamente un sistema inalámbrico utilizando una aplicación móvil, un Arduino y un módulo HC-05 para la transmisión de datos mediante Bluetooth. Este sistema permitió controlar el encendido y apagado de un LED de manera remota, lo que demostró la efectividad de la comunicación entre los dispositivos.

Adicionalmente, se identificó la importancia de mantener una coherencia entre el diseño de la aplicación y el código del Arduino. Resultó ser más eficiente desarrollar ambos componentes simultáneamente, ya que esto asegura que los datos enviados desde la aplicación sean procesados correctamente por el microcontrolador, minimizando posibles errores de comunicación y optimizando el rendimiento del sistema.
